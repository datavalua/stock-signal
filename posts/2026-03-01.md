**토스 증권 시그널 클론 코딩** 프로젝트가 드디어 10일 차라는 중대한 이정표에 도달했습니다. 어제 크롤러에 고한도 모델을 전면 도입하여 안정성을 확보한 데 이어, 오늘은 역할을 완전히 분리하여 성능을 극대화하는 '모델 이원화 전략'과 구조 개편을 진행했습니다. 10일간의 여정을 돌아보며 마주한 예기치 못한 트러블슈팅과 성능 최적화의 기록입니다.

## 1. 구조 정리와 뜻밖의 배포 오류 (Troubleshooting)

프로젝트가 커짐에 따라 코드의 유지보수성을 높이기 위해 디렉터리 구조를 `backend/`와 `data/` 등으로 체계적으로 정리했습니다. 그런데 여기서 예상치 못한 복병을 만났습니다.

### 문제: "The main module file does not exist"
구조를 정리하고 나니 멀쩡히 돌아가던 Streamlit Cloud에서 배포 오류가 발생했습니다. Streamlit Cloud는 기본적으로 저장소 루트에 있는 `app.py`를 실행 파일로 기대하는데, 내부 구조를 바꾸면서 경로를 찾지 못하게 된 것입니다.

### 해결: 유연한 경로 로직 도입
`app.py`를 다시 루트로 이동시키되, 내부에서 `backend` 모듈을 불러올 때 경로 이슈가 발생하지 않도록 로직을 보강했습니다.

```python
# app.py 상단의 유연한 경로 로직
import os, sys
sys.path.append(os.path.dirname(__file__))
from backend import crawler
```

기존의 경직된 상대 경로 대신 파일 위치를 기반으로 한 동적 경로 추가 방식을 사용하여, 로컬과 클라우드 배포 환경 어디에서나 모듈 참조 오류 없이 안정적으로 작동하도록 만들었습니다.

## 2. 시그널 품질의 복원: '개별 요약'으로의 회귀

최근 성능 향상을 위해 도입했던 '배치 요약(Batch Summarization)'이 오히려 독이 되었습니다. AI의 주의력(Attention)이 여러 종목으로 분산되면서 요약의 구체성이 떨어지는 부작용이 발견되었습니다. 

오늘 과감히 **개별 요약 방식**으로 회귀했습니다. 각 종목당 최대 1500자의 뉴스 컨텍스트를 온전히 할당하여 Gemini가 변동의 핵심 원인을 더 날카롭게 파헤칠 수 있도록 했습니다. 결과적으로 '수급 변화' 같은 평이한 설명 대신, 기업의 구체적인 실적이나 수주 소식을 담은 고품질 시그널 리포트가 다시 생성되기 시작했습니다.

## 3. 메타데이터 부트스트랩 스크립트 독립화 (bootstrap_metadata.py)

매일 반복해서 생성되는 시그널 뉴스 요약 데이터 말고도, 종목 자체의 성격을 규정해 주는 고정적인 정보들이 필요했습니다. 예를 들어 삼성전자의 "주요 산업 분야(industry)"와 "관련 기업/경쟁사(peers)" 같은 정보는 매일 바뀔 내용이 아닙니다. 

크롤러에 이 작업을 묶어두는 것은 엄청난 비효율이라고 판단하여, 이를 파일 한 개로 독립시킨 **`bootstrap_metadata.py`** 구성 스크립트를 작성했습니다. 특히, 이 스크립트는 매일 도는 크롤러와 달리 어쩌다 한 번, 혹은 신규 종목이 추가될 때만 돌아가면 됩니다.  따라서 속도와 쿼터(RPD) 한도에 구애받지 않고, 구글의 AI 모델 중 성능과 추론 품질이 가장 뛰어난 **`gemini-2.5-pro` 모델을 전격 도입**하여 고품질의 `stock_metadata.json` 파일을 정밀하게 빚어낼 수 있게 되었습니다.

## 4. 할당량(Quota) 방어 작전: Tiered Smart Skip

GitHub Actions를 통해 빈번한 크롤링을 수행하다 보니, 새로운 뉴스가 없음에도 중복해서 API를 호출하는 쿼터 낭비 문제가 대두되었습니다. 이를 해결하기 위해 **'계층형 스킵 로직(Tiered Skip Logic)'**을 구현했습니다.

- **Tier 1 (Instant Skip - Heuristic)**: 기사 목록을 스크래핑한 직후, 가장 최신 기사의 헤드라인(Title)과 URL을 이전 크롤링 사이클에서 저장했던 JSON 데이터의 최신 기사와 1:1로 비교합니다. 만약 완전히 동일하다면, 요약문을 그대로 재사용(캐싱)하여 다음 종목으로 넘어갑니다.
- **Tier 2 (AI-Assisted Skip)**: Tier 1 방어선을 통과했더라도(새로운 자잘한 기사가 떴더라도), 1차적으로 AI가 수집된 전체 기사 목록 중 *가장 주가 변동에 큰 영향을 미쳤을 법한 핵심 기사 1개를 선정*하게 합니다. 이 핵심 기사가 이미 어제 쓰였던 기사라면 비용이 큰 AI '종합 요약' API를 호출하지 않고 과감히 생략합니다.

이러한 지능형 시스템 덕분에, 아무리 자주 크롤링 루틴이 돌아도 실제 API 호출은 '주가에 영향을 줄 만한 진짜 새로운 정보'가 떴을 때만 발생하게 되었습니다. 효율적인 아키텍처 설계가 인프라 비용을 어떻게 절감할 수 있는지 여실히 보여주는 사례였습니다.
