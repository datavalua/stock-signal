## 자동화의 핵심: GitHub Actions 기반의 서버리스 연동

뉴스는 정보의 밀도가 높지만 동시에 노이즈가 매우 많습니다. 거래량이 실린 급등 종목의 경우 하루에도 수십 건의 기사가 발행됩니다. 그중에는 주가에 직접적인 영향을 준 결정적 공시나 소식도 있지만, 의례적인 시황 정리나 종목 명칭만 언급된 연관성 낮은 기사들도 다수 존재합니다. 이러한 원시 데이터를 식별 과정 없이 AI 모델에 전달하면 요약의 정확도가 현저히 떨어지게 됩니다. "매수세가 유입되어 주가가 올랐다"는 식의 무의미한 순환논리에 빠지기 쉽기 때문입니다.

이러한 현상을 방어하기 위해 뉴스 크롤링 로직을 한층 정교하게 설계했습니다. BeautifulSoup을 사용하여 기사의 제목뿐만 아니라 본문의 첫 세 문장을 집중적으로 수집했습니다. 기사 작성 방식의 특성상 핵심 사실관계는 항상 도입부에 위치하므로, 이 세 문장만으로도 AI가 문맥을 파악하기에 충분한 정보량을 확보할 수 있습니다. 또한 수집된 기사 중 가장 영향력 있는 기사를 우선순위에 따라 선정하는 알고리즘(`select_impactful_article`)을 도입했습니다.

## 통합 테스트: 요약 파이프라인의 완성도 검증

개발 과정 전반을 검증하기 위해 작성한 `tests/test_summary.py` 통합 테스트 스크립트는 전체 파이프라인의 완성도를 높이는 데 기여했습니다. 수집, 선별, 요약으로 이어지는 단계별 로직이 예상대로 작동하는지를 한눈에 확인할 수 있었습니다.

```python
# tests/test_summary.py - AI 요약 파이프라인 통합 테스트 코드
import crawler

symbol = "011200" # HMM 종목 예시
name = "HMM"
date_str = "2026-02-21" # 현재 시점을 기준으로 테스트 수행

print("단계 1: 투자자 동향 데이터 확보")
investor_data = crawler.get_investor_data(symbol, date_str)

print("단계 2: 뉴스 데이터 크롤링 및 필터링")
articles = crawler.scrape_naver_news(symbol, name, date_str)
if articles:
    # 지표에 기반하여 가장 신빙성 있는 기사 인덱스 추출
    best_idx = crawler.select_impactful_article(name, articles, 5.0)
    target = articles[best_idx]
    target['content'] = crawler.scrape_article_content(target['url'])

print("단계 3: Gemini API를 통한 최종 요약")
summary = crawler.generate_summary(name, articles, 5.0, best_idx, investor_data)

print("\n--- 파싱 및 요약 결과 보고 ---")
print(summary)
```

## 프롬프트 엔지니어링: 페르소나와 제약 조건 부여

프롬프트 엔지니어링 측면에서도 많은 개선이 있었습니다. 단순히 "요약하라"는 명령 대신, "전문 투자자가 3초 이내에 핵심을 파악할 수 있도록 팩트 중심의 한 문장으로 정리하라"는 구체적인 페르소나와 제약을 부여했습니다. 또한 시황 흐름과 무관한 해당 기업만의 고유 이슈를 특정하도록 유도했습니다. 이를 통해 "HBM3E 관련 독점 계약 체결 및 대규모 양산 기대감"과 같이 구체적이고 전문적인 통찰이 담긴 결과물을 얻을 수 있었습니다.

## 모델 성능 최적화: Gemini 1.5 Flash의 활용

Gemini 1.5 Flash 모델은 대량의 뉴스 텍스트를 빠르게 처리하는 데 매우 효율적이었습니다. 모델의 전반적인 응답 속도가 빨라 실시간성이 중요한 주식 데이터 처리 분야에서 강점을 보였습니다. 

데이터 저장 시 발생했던 한글 인코딩 깨짐 현상은 `json.dump` 수행 시 `ensure_ascii=False` 파라미터를 명시하여 해결했습니다. 데이터가 유니코드 형태로 저장되면 가독성이 떨어지므로, 서비스 운영 관점에서 필수적인 조치였습니다. 

이제 시스템은 주가 변동의 근본 원인을 추론할 수 있는 수준에 도달했습니다. 축적된 데이터의 품질이 안정화되었음을 확인했으며, 내일부터는 요약되는 정보의 문체와 다양성을 개선하여 지식 창고로서의 가치를 높이는 데 전념할 계획입니다.
