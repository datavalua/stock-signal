"사용자에게는 평온함을, 관리자에게는 강력함을."

프로토타입을 운영하다 보면 가장 큰 난관 중 하나가 바로 **'데이터의 영속성'**입니다. Streamlit Cloud는 가볍고 빠르지만, 서버가 재부팅되면 로컬에서 실행한 크롤링 결과가 사라지는 휘발성 환경이라는 한계가 있었습니다. 오늘은 이 한계를 완전히 깨부수고, 관리자 페이지에서 클릭 한 번으로 전 세계 서버(GitHub Actions)를 움직이는 '원격 제어 시스템'을 구축했습니다.

## 1. Streamlit에서 GitHub Actions를 깨우다

이제 수동 크롤링을 실행할 때 "내 컴퓨터"나 "임시 서버"에서 돌리지 않습니다. 관리자 페이지의 버튼을 누르면 즉시 GitHub API를 호출하여 **GitHub Actions 워크플로우를 원격으로 트리거**합니다.

이 마법 같은 연결을 통해 다음과 같은 이점을 얻었습니다.
- **영구 저장**: 수동으로 실행한 결과가 자동으로 GitHub 저장소에 커밋되어 영구히 보존됩니다.
- **환경 일치**: 자동 스케줄과 수동 실행이 동일한 환경(GitHub Runner)에서 돌아가므로 데이터 품질이 완벽하게 일치합니다.

## 2. 보안과 효율의 조화: 지연 로딩(Lazy Loading)

보안은 타협할 수 없는 영역입니다. `GEMINI_API_KEY` 같은 민감한 정보를 Streamlit Cloud와 GitHub 환경 모두에서 안전하게 쓸 수 있도록 API 로딩 로직을 전면 개편했습니다. 앱이 시작될 때가 아니라, 실제 AI 요약이 필요한 그 순간에만 키를 찾아오는 '지연 로딩' 방식을 도입하여 더 유연하고 안전한 구조를 만들었습니다.

## 3. 스마트한 시간 관리: RateLimiter 도입

AI 호출이 많아지면서 구글의 호출 제한(RPM)에 걸리는 문제가 발생했습니다. 무조건 길게 쉬는 대신, 1분당 허용량을 초 단위로 계산하여 **"가능한 한 가장 빠르게"** 다음 작업을 이어가는 스마트 레이트 리미터를 구현했습니다. 덕분에 크롤링 대기 시간은 줄이면서도 안정성은 극대화되었습니다.

오늘의 작업으로 'AI 주식 시그널 아카이브'는 단순한 웹 앱을 넘어, 자동화와 수동 제어가 완벽하게 조화된 탄탄한 서비스 인프라를 갖추게 되었습니다. 이제 인프라 걱정 없이 컨텐츠의 질을 높이는 데만 집중할 수 있는 환경이 마련되었습니다!
