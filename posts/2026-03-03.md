글로벌 서비스의 디테일을 완성하기 위해 **시차(Timezone)** 문제와 **번역** 품질을 개선한 12일차 기록입니다. 클라우드 서버의 UTC 시간을 **pytz** 라이브러리로 교정하여 정확한 **KST** 업데이트 시간을 구현했으며, 영문 뉴스를 직역이 아닌 자연스러운 한국어 문장으로 요약하는 **프롬프트 엔지니어링** 전략을 통해 현지화 완성도를 한 차원 높였습니다.

시그널 앱에 접속하면 "마지막 업데이트 시간"이 표기되는데, 이상하게도 이 시간이 실제 한국 시간보다 항상 9시간 과거를 가리키고 있었습니다. 

원인은 GitHub Actions와 Streamlit Cloud의 기본 시스템 시간이 **UTC(협정 세계시)**로 설정되어 있기 때문이었습니다. 파이썬의 기본 `datetime.now()`를 사용하면 서버의 로컬 시간을 불러오는데, 클라우드 환경에서는 이것이 한국 시간(KST)이 아니었던 것이죠.

이를 해결하기 위해 `pytz` 라이브러리와 `datetime.timedelta`를 활용하여 어플리케이션 전역의 시간 기준점을 명시적으로 KST(`Asia/Seoul`)로 고정했습니다.

```python
# app.py 의 업데이트 시간 표기 로직 (KST 강제 변환)
from datetime import datetime
import pytz

def get_kst_time_str(date_str):
    # UTC 기준의 시스템 시간을 명시적으로 KST로 변환하여 사용자에게 제공
    kst = pytz.timezone('Asia/Seoul')
    now = datetime.now(kst)
    return now.strftime("%Y-%m-%d %H:%M:%S (KST)")
```
이제 사용자들은 화면 최상단에서 직관적이고 정확한 한국 기준 업데이트 시간을 확인할 수 있게 되었습니다.

### 2. 반쪽짜리 정보 극복: 미국 주식 완벽한 한글 요약

미국 주식(US Market) 기능을 오픈했을 때, 원본 영문 기사를 그대로 Gemini AI에게 "요약해 줘(Summarize this)"라고만 지시했더니, AI가 요약은 잘해주지만 **답변을 영어로 반환**하거나, 한국어로 번역하더라도 **직역투의 매우 어색한 문장**을 내놓는 문제가 빈번했습니다.

예를 들어 테슬라의 주가 변동 원인을 "The stock went up because of..." 처럼 영어로 뱉거나, "일론 머스크가 트윗을 하였기 때문에 주식은 상승의 영향을 겪었습니다" 와 같이 기계적인 번역이 출력되는 식입니다. 

이를 해결하기 위해 프롬프트 엔지니어링을 다시 한번 정교하게 손보았습니다.

- **명시적 출력 언어 고정**: 시스템 프롬프트 최상단에 `[LANGUAGE REQUIREMENT: MUST ANSWER IN KOREAN ONLY]` 라는 절대 규칙을 삽입했습니다.
- **번역과 요약의 동시 수행**: 단순히 '번역'을 시키는 것이 아니라 전문 금융 애널리스트의 페르소나를 부여하여, "영문 기사 안에서 월스트리트의 핵심 변동 동인을 파악한 뒤 한국의 개인 투자자가 읽기 편한 자연스러운 경제 기사체로 요약하라"고 지시했습니다.

이 작은 프롬프트 교정 하나로 미국 주식 시그널의 가독성이 국내 주식 시그널과 완벽하게 동일한 수준으로 올라왔습니다. 기술은 보이지 않게 작동할 때 가장 위대하다고 했던가요. KST 시계와 매끄러운 한국어 번역 로직이 더해지면서 서비스의 완성도와 '현지화(Localization)'가 한 차원 높아진 하루였습니다. 

내일은 개별 종목을 넘어, 전반적인 시장의 온도를 읽을 수 있는 **'시장 지수(Index)' AI 요약 엔진**을 구축하는 여정을 기록하겠습니다.
