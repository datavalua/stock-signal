"코딩은 기세다."

**토스 증권 시그널 클론 코딩** 프로젝트가 중반부에 접어들며 시스템이 점차 복잡해짐에 따라 예상치 못한 운영 이슈들이 발생하기 시작했습니다. 오늘은 서비스의 핵심인 AI 요약 기능이 완전히 중단되는 기술적 장애를 극복하고, 장기적인 운영 안정성을 확보하기 위한 기능을 구현하는 데 많은 시간을 할애했습니다.

## 위기 상황: 유출된 API 키와 서비스 중단

발단은 아침 모니터링 중 발견된 대규모 데이터 누락이었습니다. 모든 종목의 시그널 요약이 "생성 오류"라는 안내 메시지로 치환되어 출력되고 있었습니다. 원인 파악을 위해 GitHub Actions의 실행 로그를 전수 조사한 결과, `INVALID_ARGUMENT: API key has been invalidated because it was leaked.`라는 치명적인 에러 메시지를 확인했습니다. 이는 개발 초기 단계에서 API 키를 소스 코드 내에 포함하여 공개 저장소에 푸시했던 과거 이력 때문에 구글의 보안 엔진이 사후에 해당 키를 강제 만료시킨 것이었습니다.

## 보안 강화: GitHub Secrets와 .env 관리 체계

키 관리에 대한 기본 수칙을 간과했던 대가는 컸지만, 이를 통해 보안 프로세스를 전면 재정립하는 기회를 가질 수 있었습니다. 새롭게 발급받은 API 키는 GitHub Secrets 환경 변수 시스템에 엄격히 격리하여 등록했으며, 로컬 개발 환경에서도 `.env` 파일과 `python-dotenv` 라이브러리를 사용하여 민감 정보가 소스 코드에 직접 노출되지 않도록 구조화했습니다. 보안은 번거로운 절차를 요구하지만, 이는 안정적인 서비스 운영을 위한 필수적인 투자임을 다시 한번 확인했습니다.

## 운영 도구: Streamlit 기반 Admin 페이지 구축

또한, 자동화된 AI 분석이 가진 한계를 보완하기 위해 '관리자 페이지(Admin)'를 신규 구축했습니다. AI는 대량의 데이터를 신속하게 처리할 수 있다는 장점이 있지만, 때로는 복잡한 중의적 의미를 가진 뉴스를 오해하거나 부적절한 표현을 사용하는 경우가 발생합니다. 개발자가 코드 수준에서 일일이 대응하기보다는 웹 인터페이스를 통해 즉시 개입할 수 있는 운영 도구가 필요하다고 판단했습니다.

관리자 페이지는 Streamlit의 `st.session_state`를 활용하여 가변적인 로그인 세션을 관리하도록 설계했습니다. 별도의 복잡한 인증 서버 구축 없이도 암호 확인 과정을 거쳐 관리자 권한을 부여하는 방식을 채택하여 개발 속도를 높였습니다.

위 이미지는 프로젝트 후반부에 완성된 관리자 전용 제어 화면의 모습입니다. 5일차인 당시에는 이러한 미려한 디자인보다는 데이터의 수정과 삭제라는 본연의 기능 로직을 구현하는 데 주력했습니다.

비록 API 키 유출 사고로 인해 잠시 주춤했으나, 결과적으로는 더 견고한 보안 환경과 운영 효율성을 확보하게 되었습니다. 내일은 파편화된 크롤러들의 실행 성능을 최적화하고, 어떤 PC 환경에서도 동일한 개발 경험을 제공하는 컨테이너 환경 구축에 대해 다룰 예정입니다.
